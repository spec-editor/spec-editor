Why Spec Editor

Beyond a single LLM prompt

Raw LLMs produce superficial bullet points. Spec Editor orchestrates multi-agent debate to build deeply connected, structured specifications.

Multi-agent debate

AI agents and an orchestrator discuss your requirements from different angles. They challenge each other, fill gaps, and build consensus — eliminating single-perspective blind spots.

Methodology-driven structure

Not a flat list. Modules, scenarios, entities, UI, metrics — all connected with typed relationships. Waterfall by default. More methodologies available as packs.

MCP for coding assistants

Connect to Claude Code, Cursor, or Zed via MCP. Your AI coding agent gets the full specification as context — generating code aligned with your requirements.

Quick Start

Three steps to structured specs

From messy documents to a connected specification your AI can use.

1

Add your requirements

echo "Users log in via email..." > source/auth.md

Drop Markdown, TXT, or chat logs into source/. The pipeline extracts and classifies requirements.

2

Run the agents

spec-editor run

Multi-agent debate produces a structured specification in aspects/ — 8 aspects, typed relationships.

3

Connect your coding agent

spec-editor mcp &

Start the MCP server. One JSON config block — Claude Code, Cursor, or Zed reads your spec.
