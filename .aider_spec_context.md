### MOD-001: MCP Server Module
Handles JSON-RPC 2.0 MCP server with two transports: stdio (stdin/stdout) and HTTP (port 8088). Provides specification tools (read_element, write_element, add_relationship, etc.), code tools (search_code, verify_implements), and export tools (generate_diagram, git_history). Stateless HTTP handler with optional --read-only mode. SSE hub for real-time events. Binds to 127.0.0.1 only for security.

  consists_of: CA-001, CA-005, COM-001
  depends_on: MOD-004, MOD-005, COM-001
  tested_by: TC-001

### MOD-004: Storage Layer Module
FilesystemStorage: elements stored as .md files with YAML frontmatter in aspects/{aspect_name}/{ID}.md. Index rebuilt on init. Merge-on-write semantics (union of relationships, children, prefer longer content). Atomic writes via tempfile.mkstemp + os.replace. Reference validation checks relationship target existence. DryRunStorage wraps two FilesystemStorage instances for testing without modification.

  consists_of: CA-002
  depends_on: MOD-005
  tested_by: TC-004
